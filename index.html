Don't worry

Been there, done that myself.

Do not fly by

As much as you might think your feedback is important, if it wasn't asked, don't give it.

Style is not up for discussion

Have automated tools that make the style conformant to whatever rules you have. If you're making comments about code style in a PR the process has failed.

Have static analysis tools available

  • Checkstyle
  • go vet
  • rubocop
  • jslint

If you don't understand, ask

If the change is unclear or follows a pattern that is unknown to you, ask for an explanation first before asking for it to be done in a different way.

Call out good stuff

Found something cool? Learned something new? Call it out, it's great to get a +1 on reviews.

Do not nitpick

If the only comment you could possibly make has no real effect on the final result, do not make it. You're not getting points for participating.

Be nice

I mean, be an empathetic human being, you're not fighting the other person, you are all responsible for the codebase together, provide useful, constructive feedback and learn with the code that's being provided.
